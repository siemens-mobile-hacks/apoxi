USB Overview

1 Introduction

USB stands for Universal Serial Bus. It is used as a possible transport protocol by OBEX, which is a standard protocol used to exchange objects consisting of headers and data between a client and a server. It follows a request-response mechanism. In APOXI, Obex uses Bluetooth, IrDA and USB as the possible transport protocols. Tthe USB OBEX transport layer is the glue between the APOXI OBEX module and the Emsys Obex software.

2 Overview

The USB Obex layer in APOXI acts as the transport protocol and provides an abstraction over the Emsys Obex implementation.

ApoxiObex module allows applications to perform client-server communication as per the OBEX standard for the available transport protocols.

UsbObex provides the UsbObex transport implementation for the ApoxiObex module. The implementation of this module lies in /Apoxi/AddOns/Usb/Obex folder. UsbObex feature can be enabled or disabled using the compiler flag 'USB_OBEX'.

UsbTransport is independent of the UsbObex implementation. It provides the USB device specific information to its subscribers.

EmsysObex module provides the protocol implementation of OBEX over USB.

UsbApplication is a Mmi application which displays USB device connection information.

ObexAwareApp is any application that uses ApoxiObex for Obex functionality.

ObexApp is a Mmi application for the user to select one of the available transports, viz., Bluetooth, IrDA, USB.
