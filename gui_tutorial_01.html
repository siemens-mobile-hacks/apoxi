1 Introduction

The features of the APOXI-GUI are:

  • Completely written in C++: All characteristics of an object orientated programming language, like polymorphism, inheritance and encapsulation, are used in a strict manner and lead to an easy to use, but powerful framework.
  • Event-based communication: All actions are triggered by events. The communication between the components is done by messages. This proven approach offers strong benefits in comparison to state-machines.
  • Integrated window-system: Support of overlapping and nested windows. Position, size and attributes of windows can be changed dynamically.
  • Controls for menus, radiobuttons, checkboxes, listboxes, editors, ... which can be integrated within windows in an easy manner.
  • Dynamically changeable look and feel: The look and feel of the menus, windows, buttons and other controls can be altered during runtime using the theme manager.
  • Multilingual support: The full integration of Unicode and the language manager facilitates a switch of languages at any time.
  • Image support: Encoders and decoders for a broad range of image formats are available.
  • Hardware independent graphical device interface: All graphical operations are hardware independent, but achieve optimal performance by relying on a hardware specification files represented as Devices.
