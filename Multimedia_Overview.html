MultiMedia Overview

1 Introduction

The Multimedia is an APOXI-AddOn. It contains classes for decoding JPEG, PNG, BMP, WBMP and GIF images. Additionally a multimedia editor is offered where animations and sound objects can be embedded and edited inside a text. Also it is possible to encode JPEG images in a synchronous and asynchronous manner.

2 MultiMediaEditor

The multimedia-editor can be used for displaying and editing texts in combination with bitmaps, animations and sounds. The editor itself does not care about the format when receiving or sending the data (e. g. EMS or MMS), but offers an easy interface to feed the editor-document with the corresponding data and retrieve the modified data.

MultiMediaEditor offers a multi-media-editor with the following features:

  • Suitable for the input-mappers for numbers, phonenumbers, multitap and predictive input like T9
  • Vertical scrolling including an optional position-indicator
  • Overwrite/insert mode
  • Allows embedding a variable number of images, animations, sound-objects and other customized editor-items
  • Support of paragraphs, which may aligned left, centered or right
  • Support of rich-text:
    • Bold, italic, underlined, strikethrough
    • Variable text-color and text-background-color
    • Variable font
  • Read-only mode which allows triggering actions on images, animations and sound-objects

The multimedia-editor uses the multimedia-editor-document for accessing and modifying the data. The multimedia-document can contain a variable number of compound-Items.

2.1 Compound-Items

A compound-Item contains a variable number of Items like characters or bitmaps. APOXI offers the following compound-Items:

  • Text-Compound-Item: Can contain a text of a variable length
  • Date-CompoundI-tem: Contains a dat
  • Time-Compound-Item: Contains a time
  • Ip-Address-Compound-Item: Contains an IP-address

In the multimedia-package of APOXI-Add-Ons the following compound-Items are available:

  • Rich-Text-Compound-Item: Can contain a text of a variable length, where the text can have a font, foreground-color, background-color and a text-attribute (derived from TextCompoundItem).
  • Paragraph-Compound-Item: Represents a text-paragraph, which can have a horizontal alignment (default, left, centered, right) and contains a variable number of compound-Items.
  • General-Compound-Item: Can contain a variable number of items which know their size themselve (e. g. an Image-Item).

The multimedia-document automatically takes care to split and merge the compound-Items when items are inserted or deleted. Hence always a minimum amount of compound-Items is assured.

2.2 Input-Method

The input-method can be any Input-Mapper offered by APOXI (e. g. MultiTab or Predictive) or an individual Input- Mapper.

2.3 Wrapping

Every compound-Item itself takes care for the wrapping. Different wrapping-modes are supported.
